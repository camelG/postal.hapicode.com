郵便番号 API — 無料の日本郵便番号・住所検索

postal.hapicode.com は、日本の郵便番号（7 桁）から住所（都道府県・市区町村・町域）を取得できる、認証不要・API キー不要の静的 JSON API です。フロントエンド向けに CORS 対応しており、GitHub Pages 上で配信されるためブラウザやサーバーからそのまま fetch できます。日本郵便の公開データを元に毎日自動更新されます。

ベース URL https://postal.hapicode.com API バージョン v6 郵便番号 — 件 最終更新 読み込み中…

特徴

認証不要 API キーやトークンは不要です。
CORS 対応 フロントエンドから直接リクエストできます。
静的 JSON CDN キャッシュが効く、サーバーレス配信です。
毎日自動更新 日本郵便の公開データから生成されます。

試してみる

7 桁の郵便番号を入力すると、実際の API レスポンスを取得できます。

エンドポイント一覧

用途 メソッド パス
API メタ情報 GET /api/index.json
郵便番号検索 GET /api/v1/{上3桁}/{下4桁}.json
都道府県一覧 GET /api/v1/regions/index.json
都道府県別 市区町村・町域 GET /api/v1/regions/{都道府県名}.json

郵便番号検索

7 桁の郵便番号を上 3 桁（ディレクトリ）と下 4 桁（ファイル名）に分けてリクエストします。

郵便番号 リクエスト URL
0010023 /api/v1/001/0023.json
5590005 /api/v1/559/0005.json
0600000 /api/v1/060/0000.json

URL の組み立て方

const zip = "5590005";
const url = `/api/v1/${zip.slice(0, 3)}/${zip.slice(3)}.json`;
// → /api/v1/559/0005.json
存在しない郵便番号 該当する JSON ファイルがない場合、HTTP 404 が返ります。

レスポンス形式

郵便番号検索は常に次の 4 フィールドを持つフラットオブジェクト（配列なし）を返します。 

{
  "prefecture": "大阪府",
  "city": "大阪市住之江区",
  "town": "西住之江",
  "address": "大阪府大阪市住之江区西住之江"
}
フィールド 型 説明
prefecture string 都道府県名
city string 市区町村名（正規化済み）
town string 町域名（正規化済み。該当なしの場合は空文字）
address string prefecture + city + town の連結

複数町域がある郵便番号

1 つの郵便番号に複数の町域が紐づく場合でも、レスポンスは 1 件のみです。日本郵便 CSV の先頭行を採用します。括弧注記（例: （小山台地））は正規化で除去されます。

住所の正規化

出力前に町域・市区町村名から括弧注記と日本郵便のプレースホルダを削除します。詳細は /api/index.json の normalization フィールドを参照してください。

変換前 変換後
大手町（次のビルを除く） 大手町
北一条西（１～１９丁目） 北一条西
伊敷町（その他） 伊敷町
以下に掲載がない場合 （空文字）

API メタ情報

GET /api/index.json でバージョン、統計、各 API のパスを確認できます。 

{

  "source": "https://www.post.japanpost.jp/service/search/zipcode/download/kogaki/zip/ken_all.zip",
  "recordCount": 124827,
  "zipcodeCount": 120682,
  "normalization": {
    "town": "remove_parentheses"
  },
  "chunkBy": "prefix3+suffix4",
  "lookupPath": "/api/v1/{prefix3}/{suffix4}.json",
  "stats": {
    "prefix3Dirs": 948,
    "shardFiles": 120682
  },
  "apis": {
    "zipLookup": "/api/v1/{prefix3}/{suffix4}.json",
    "regionsIndex": "/api/v1/regions/index.json",

フィールド 説明
version API スキーマのバージョン（現在 6）
updatedAt データ生成日時（ISO 8601, UTC）
recordCount 日本郵便 CSV の行数
zipcodeCount 生成された郵便番号 JSON ファイル数
chunkBy ファイル分割方式（prefix3+suffix4）
apis 各エンドポイントのパステンプレート

都道府県 API

都道府県単位で市区町村と町域の一覧を取得できます。フォームの都道府県・市区町村セレクトボックスの構築などに利用できます。

都道府県一覧

GET /api/v1/regions/index.json 

{
  "updatedAt": "2026-07-02T16:48:10.651Z",
  "prefectures": [
    {
      "name": "愛知県",
      "path": "/api/v1/regions/%E6%84%9B%E7%9F%A5%E7%9C%8C.json",
      "cityCount": 69
    }
  ]
}

都道府県別データ

都道府県名を URL エンコードして指定します。例: 大阪府 → /api/v1/regions/%E5%A4%A7%E9%98%AA%E5%BA%9C.json 

{
  "prefecture": "大阪府",
  "cities": {
    "大阪市住之江区": ["西住之江", "柴谷", "住之江"]
  }
}

cities のキーは市区町村名、値は町域名の配列です。町域名も郵便番号検索と同様に正規化されています。

利用例

JavaScript

const API_BASE = "https://postal.hapicode.com";

export async function lookupZipcode(zipInput) {
  const zip = zipInput.replace(/\D/g, "");
  if (zip.length !== 7) return null;

  const res = await fetch(
    `${API_BASE}/api/v1/${zip.slice(0, 3)}/${zip.slice(3)}.json`,
  );
  if (!res.ok) return null;

  return res.json();
}

// 使用例
const address = await lookupZipcode("559-0005");
// {
//   prefecture: "大阪府",
//   city: "大阪市住之江区",
//   town: "西住之江",
//   address: "大阪府大阪市住之江区西住之江"
// }

curl

curl -s https://postal.hapicode.com/api/v1/559/0005.json
curl -s https://postal.hapicode.com/api/index.json
curl -s https://postal.hapicode.com/api/v1/regions/index.json

データと更新

  • 出典: 日本郵便 全国地方公共団体コード（町域）データ （ken_all.zip）
  • 更新頻度: 毎日 0:00 JST に postal-backend.hapicode.com の GitHub Actions により自動生成・デプロイ
  • 件数: 120,682 件の郵便番号（zipcodeCount）、948 ディレクトリ（上 3 桁）
  • 更新日時: /api/index.json の updatedAt を参照

利用上の注意

  • 本 API は無償で提供していますが、可用性や SLA は保証しません。本番サービスではフォールバックやキャッシュの導入を推奨します。
  • 住所データの正確性は日本郵便の公開データに依存します。利用・再配布については 日本郵便の利用規約 に従ってください。
  • 過度なリクエストは控えてください。静的ファイル配信のため CDN キャッシュが効きます。

ライセンス

住所データの利用条件は 日本郵便の利用規約 に従ってください。API 配信形式（本サイトの JSON 構造）については MIT ライセンスの下で提供しています。
